- 대상: `index.html` v9.47.0
- 일자: 2026-07-17

1. 모바일 전역 스와이프와 롱프레스·내부 가로 조작이 충돌할 수 있다.

## 페르소나 4 — 모바일 한 손 사용자

### 환경
- 360×800 안드로이드
- 한 손 조작
- 목표: 태그 선택, 에디터 이동, 결과 복사

### 실행 흐름
1. 태그 패널에서 선택
2. 좌우 스와이프로 에디터 이동
3. 태그 롱프레스
4. 결과 패널 이동
5. 복사

### 결과
- 입력 요소, 인페인트 마스크, 라이트박스에서는 전역 패널 스와이프를 피하도록 방어가 있다.
- 수평 방향이 확정된 뒤 `preventDefault()`하는 흐름도 기본적인 세로 스크롤 보호에 적절하다.

### 문제
- 전역 `document` 스와이프가 태그 목록의 내부 가로 스크롤, 커스텀 슬라이더, 롱프레스 대상에서 시작할 수 있다.
- 롱프레스는 12px 이동 시 취소되지만 전역 스와이프는 8px에서 수평 잠금된다. 사용자가 롱프레스를 시도하다 약간 움직이면 메뉴 대신 패널이 밀릴 수 있다.
- `touchcancel`이 발생했을 때 `swDragging`이 false이면 시작 좌표가 남는 경로가 있어 다음 touchmove가 이전 터치의 좌표를 재사용할 가능성이 있다.

### 판정
- 부분 성공
- P1: 전역 스와이프 제외 대상을 명시하고 `touchcancel`에서 상태를 조건 없이 초기화

# 통합 발견 사항

## P1 — 우선 수정

### 1. 모바일 제스처 충돌
- 전역 패널 스와이프, 롱프레스, 내부 가로 조작이 각각 별도 document listener로 동작한다.
- 하나의 공통 gesture lock을 두고 `panel-swipe`, `long-press`, `canvas`, `inner-scroll` 중 하나만 소유하도록 해야 한다.

### 2. 백업 복원의 원자성 부족
- localStorage 순차 저장 중 실패하면 부분 복원이 가능하다.
- 복원 성공 여부를 키별로 수집하고 실패 시 기존 상태를 보존해야 한다.

## P2 — 다음 수정 묶음

1. 갤러리 URL 기반 메타 키 충돌
2. 고정 접두 태그의 가중치 정규화 누락
3. 마지막 라이브 프레임 Object URL 정리
4. 인페인트 전체 영역의 패널 스와이프 차단 명시화

4. 동일 URL 이미지 2개, 서로 다른 시드·프롬프트 → 각각 독립 메타 유지
5. `(score_9:1.1)` 본문 + `score_9` 고정 접두 → 정책에 따른 단일 태그 출력
6. 라이브 프레임 1회 수신 후 작업 완료 → Object URL revoke 확인

# 최종 결론

프롬포트랩의 일반적인 PC 사용 흐름과 ComfyUI 상태 격리는 최근 수정으로 상당히 안정됐다. 현재 위험은 핵심 생성 로직보다 모바일 제스처 중재와 대량 localStorage 작업처럼 여러 기능이 한꺼번에 만나는 경계 영역에 집중돼 있다.

다음 수정 순서는 다음이 적절하다.

1. 모바일 gesture lock
2. 백업 복원 원자성
3. 갤러리 데이터 모델을 고유 ID 기반으로 변경
4. Object URL·가중치 중복 등 잔여 정리

이 보고서는 실제 코드 경로를 기반으로 한 페르소나 실행 결과이며, ComfyUI 서버 출력 품질과 모바일 브라우저의 픽셀·제스처 반응은 배포 URL 또는 로컬 실행 환경이 준비되면 별도 실기 E2E로 보완해야 한다.
